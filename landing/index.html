Features

Everything your agent team needs

Built for teams running OpenClaw who need visibility, control, and accountability over their AI workforce.

📋

Kanban Board

9 columns — one per agent plus the User column. Tasks move automatically as agents progress. See your entire pipeline at a glance.

⚡

Real-Time WebSocket

All connected browsers stay in sync instantly. No refresh. Task state propagates from OpenClaw gateway events in milliseconds.

📊

6-Widget Dashboard

Activities, Success Rate, Tokens by Provider, Most Used Agent, Issues, and Live Logs. Full observability without leaving the page.

🧠

WBS Engine

Every task auto-generates a 4-item Work Breakdown Structure. Track subtask completion per agent. Know exactly where each task stands.

🛡️

Full Audit Log

Every status change, agent action, and token spend is logged. Filter by agent, level, or time. Export-ready for compliance.

🔌

Plug-and-Play

Add 5 lines to your OpenClaw config, restart the gateway, and DashR starts receiving live agent events. No code changes to your agents.

💰

Cost Tracking

Token usage and cost per task, per agent, per provider. Know exactly what each workload costs across Anthropic, OpenAI, and Google.

🤖

Connie Orchestration

Smart routing logic automatically assigns tasks to the right agent based on prompt analysis. Or assign manually via dropdown.

🗄️

SQLite Zero-Config DB

Runs with zero database setup. Single file, instant start. Upgrade path to PostgreSQL when you need multi-user or cloud persistence.

How it works

From task to delivery, in four steps

DashR sits between your team and your OpenClaw agents, tracking every move.

1

Create a Task

Write a prompt, pick an agent, choose a model. DashR generates the task ID and WBS.

2

Connie Routes

Connie analyzes the task and routes it to the right specialist — or you assign manually.

3

Agents Execute

OpenClaw runs the agent session. DashR receives live events via WebSocket and hooks.

4

Track & Deliver

Watch the Kanban board update in real time. Tasks move to Done when agents complete them.

Integration

Plug into any OpenClaw
in 5 lines

DashR hooks directly into OpenClaw's event system. Add this to your config and restart.

// ~/.openclaw/openclaw.json { "hooks": { "enabled": true, "token": "${OPENCLAW_HOOKS_TOKEN}", "endpoints": [{ "url": "http://localhost:3000/api/hooks/openclaw", "events": ["session:update", "session:complete", "session:waiting"] }] } }
Then: openclaw gateway restart
Tech Stack

Simple by design.
Powerful by necessity.

No frameworks, no build steps, no Docker required for development. Just Node.js.

Backend
Node.js 20 + Express
Same runtime as OpenClaw. CommonJS for zero-transpile development. REST API + WebSocket on a single port.
Database
SQLite → PostgreSQL
Zero-config SQLite for MVP. Knex query builder gives you a one-line migration path to PostgreSQL when you need it.
Real-Time
WebSocket (ws)
Native WebSocket library — same protocol OpenClaw uses internally. All browsers stay in sync without polling.
Frontend
Vanilla JS + Chart.js
No React, no Vite, no bundler. The dashboard is a single HTML file served as static. Fast, inspectable, hackable.
OpenClaw Bridge
CLI subprocess + WS client
Invokes agents via openclaw CLI subprocess. Subscribes to gateway events via WebSocket. Receives callbacks via hooks.
Deployment
Docker / PM2 / Bare Node
Run anywhere. Docker Compose for production. PM2 for daemon mode. Or just npm start for instant local use.
